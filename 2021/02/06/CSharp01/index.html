核心 C#

2021-02-06 C#

笔记都是基于 C# 7.0 和 .NET Core 2.0，会有一部分新特性

只记录相对于重要的地方，具体请翻书

核心 C#

[TOC]

1. 从 Hello World 开始

  • C# 区分大小写
  • using static，打开类的静态成员
  • System 名称空间，包含了最常用的.NET类

2. 变量

  • 安全性的一种体现，C# 编译器将未初始化看作一种错误，类和结构的变量会有默认值，方法的局部变量必须显式初始化
  • 类型推断：var关键字
  • Type ageType = age.GetType();

常量

const int a = 10;

  • 声明时初始化，指定值后不能改写，声明之外赋值编译器报错
  • 不能用变量中的值初始化，除非是只读字段
  • 隐式静态（默认静态），但不允许显式声明 static

3. 预定义数据类型

值类型（堆栈）

  • 变量赋给变量本质是值复制，堆栈中有两个变量
  • 自己定义的类型通常是引用类型，但结构是值类型

sbyte, short, int, long, byte, ushort, uint, ulong（前 4 个有符号整数，后 4 个无符号整数，8bit，16bit，32bit，64bit）

  • 数字分隔符（C# 7.0 开始支持），提高可读性，编译器自动忽略，例：long ll = 0x123_4567_89ab_cdef;
  • 二进制赋值给整数（C# 7.0 开始支持），0b 前缀，例：uint binary1 = 0b1111_1110_1101_0011_1010_1001_0001_0101;

float, double, decimal（32bit，64bit，128bit）

  • decimal 不是基本数据类型，有性能损失

true, false

  • 不能和非 0 和 0 隐式转换

char

  • 16bit 的Unicode字符

引用类型（托管堆）

  • 通过多个别名访问堆中的数据，变量赋值指向同一块堆内存

  • CLR 定期删除不能访问的对象，释放其占用的内存

String 和 Object

  • 给 string 变量赋值时会创建一个新的 string，运算符重载的结果
  • @前缀，不转义字符串中的任何字符
  • $前缀，字符串插值

.NET类型

数据类型的关键字，如int等会从编译器映射到.NET数据类型

也就是说，这些预定义的数据类型本质上是某个类的实例，从而可以直接通过这种特殊的实例来调用某些方法，如类型转换

例：声明一个int时，实际上创建了一个.NET struct:System.Int32类的实例，从而可以通过类似i.toString();的方法实现转换

存储时仍存为基本类型，只是在概念上用 C# 结构表示，没有性能损失

4. 程序控制

条件

if、switch

  • switch 语句，编译器强制将没有 break 的 case 子句（非空子句）标记为错误，也就是不允许fall through
  • case 子句为空时才可跳下一条语句

循环

一种逻辑组合

  • 嵌套名称空间以创建层次结构

using，避免输过长的类名
